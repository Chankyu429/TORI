# Code Area

## 코드 영역(Code Area)이란?

코드 영역(Code Area)은 프로그램의 실행 코드가 저장되는 메모리 영역이다.

프로그램이 실행되면 컴파일된 명령어들이 메모리에 적재되며, CPU는 이 영역의 명령어를 읽어 프로그램을 실행한다.

Code 영역은 Text 영역(Text Segment)이라고도 한다.

---

## 코드 영역의 특징

- 프로그램의 실행 코드가 저장된다.
- 읽기 전용(Read Only) 영역이다.
- 프로그램 실행 중 내용이 변경되지 않는다.
- 여러 프로세스가 공유할 수 있다.
- 프로그램 시작 시 생성되고 종료 시 제거된다.

---

## 코드 영역에 저장되는 내용

- 함수 코드
- 메서드 코드
- 반복문
- 조건문
- 연산 명령어

예:

```csharp
void Print()
{
    Console.WriteLine("Hello");
}
```

위 함수의 실행 명령어는 Code 영역에 저장된다.

---

## 코드 영역의 역할

CPU는 Code 영역에 저장된 명령어를 순서대로 읽고 실행한다.

즉, 프로그램이 어떤 작업을 수행할지 결정하는 명령들이 저장되는 공간이다.

---

## 코드 영역의 메모리 구조

```

┌─────────────┐
│   Stack     │
├─────────────┤
│    Heap     │
├─────────────┤
│    Data     │
├─────────────┤
│    Code     │
└─────────────┘

```

---

## 코드 영역의 장점

- 메모리 낭비를 줄일 수 있다.
- 프로그램의 안정성을 높인다.
- 실행 중 코드가 변경되지 않는다.
- 여러 프로세스가 동일한 코드를 공유할 수 있다.

---

## 코드 영역과 데이터 영역 비교

| 항목 | Code 영역 | Data 영역 |
|------|----------|----------|
| 저장 내용 | 실행 코드 | 전역 변수, 정적 변수 |
| 수정 가능 | 불가능 | 가능 |
| 읽기 전용 | O | X |
| 생성 시점 | 프로그램 실행 | 프로그램 실행 |
| 제거 시점 | 프로그램 종료 | 프로그램 종료 |

---

## C# 예시

```csharp
class Program
{
    static void Main()
    {
        PrintMessage();
    }

    static void PrintMessage()
    {
        Console.WriteLine("Hello World");
    }
}
```

- `Main()` 메서드 → Code 영역
- `PrintMessage()` 메서드 → Code 영역

---

## 결론

Code 영역은 프로그램의 실행 명령어가 저장되는 메모리 공간이다.

CPU는 Code 영역의 명령어를 읽어 프로그램을 실행하며, 해당 영역은 읽기 전용으로 관리되어 프로그램의 안정성과 메모리 효율성을 높여준다.
