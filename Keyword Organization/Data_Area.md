# Data Area

## 데이터 영역(Data Area)이란?

데이터 영역(Data Area)은 프로그램에서 사용하는 전역 변수와 정적 변수(static 변수)가 저장되는 메모리 영역이다.

프로그램이 시작될 때 메모리에 생성되며, 프로그램이 종료될 때까지 유지된다.

---

## 데이터 영역의 특징

- 전역 변수가 저장된다.
- 정적(static) 변수가 저장된다.
- 프로그램 시작 시 생성된다.
- 프로그램 종료 시 제거된다.
- 프로그램 실행 중 계속 유지된다.

---

## 데이터 영역에 저장되는 데이터

### 전역 변수

프로그램 전체에서 사용할 수 있는 변수이다.

### 정적 변수

static 키워드를 사용하여 선언한 변수이다.

---

## 데이터 영역의 역할

프로그램 전체에서 공유해야 하는 데이터를 저장한다.

여러 함수나 메서드에서 동일한 데이터를 사용할 수 있도록 관리하는 역할을 한다.

---

## 데이터 영역의 메모리 구조

```
┌─────────────┐
│   Stack     │
├─────────────┤
│    Heap     │
├─────────────┤
│    Data     │
├─────────────┤
│    Code     │
└─────────────┘
```

---

## C# 예시

```csharp
class Program
{
    static int count = 0;

    static void Main()
    {
        count++;
        Console.WriteLine(count);
    }
}
```

위 코드의 `count` 변수는 Data 영역에 저장된다.

---

## 데이터 영역의 장점

- 프로그램 전체에서 데이터를 공유할 수 있다.
- 프로그램 실행 중 계속 유지된다.
- 여러 함수에서 쉽게 접근할 수 있다.
- 메모리를 효율적으로 관리할 수 있다.

---

## 데이터 영역의 단점

- 너무 많이 사용하면 메모리 사용량이 증가한다.
- 전역 데이터가 많아지면 프로그램 관리가 어려워질 수 있다.
- 여러 곳에서 값을 변경할 수 있어 오류가 발생할 수 있다.

---

## Code 영역과 Data 영역 비교

| 항목 | Code 영역 | Data 영역 |
|------|----------|----------|
| 저장 내용 | 실행 코드 | 전역 변수, 정적 변수 |
| 수정 가능 | 불가능 | 가능 |
| 읽기 전용 | O | X |
| 생성 시점 | 프로그램 실행 | 프로그램 실행 |
| 제거 시점 | 프로그램 종료 | 프로그램 종료 |

---

## 활용 예시

- 게임 점수 저장
- 플레이어 수 관리
- 프로그램 설정값 저장
- 공통 데이터 관리

---

## 결론

Data 영역은 전역 변수와 정적 변수가 저장되는 메모리 공간이다.

프로그램이 실행되는 동안 계속 유지되며, 여러 함수와 메서드가 공통으로 사용하는 데이터를 관리하는 역할을 수행한다.
