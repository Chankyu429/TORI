# Data Area

## 데이터 영역(Data Area)이란?
> 데이터 영역(Data Area)은 프로그램이 실행되는 동안 계속 유지되어야 하는 데이터를 저장하는 메모리 영역이다.
> 프로그램이 시작될 때 메모리가 할당되며, 프로그램이 종료될 때까지 유지된다. 시스템 구조에 따라 초기화 여부에 따라 데이터(.data) 영역과 BSS(.bss) 영역으로 나누어 관리되기도 한다.

---

## 데이터 영역의 특징
- 프로그램 시작 시 메모리가 할당된다.
- 프로그램 종료 시 메모리가 해제된다.
- 전역 변수(Global Variable) 또는 정적 변수(Static Variable)가 저장된다.
- C# 환경에서는 구조적으로 관리형 힙(Managed Heap) 내의 특정 영역에 할당되지만, 개념적으로는 데이터 영역의 역할을 수행한다.
- 프로그램 전체에서 공유할 수 있는 데이터를 관리한다.

---

## 데이터 영역에 저장되는 데이터
* **전역 및 정적 변수(Global & Static Variable)**
> 프로그램 전체 혹은 해당 클래스 내부에서 공유되며, 인스턴스를 생성하지 않아도 접근할 수 있는 변수이다.
```csharp
class Program
{
    // 초기화된 정적 변수 (개념적 .data 영역)
    static int score = 100;

    // 초기화되지 않은 정적 변수 (개념적 .bss 영역)
    static int userCount;
}
```
**정적 클래스의 데이터**
`static` 키워드로 선언된 클래스나 변수이다.
```csharp
class Player
{
    public static int playerCount = 0;
}
```
**데이터 영역의 역할**
> 프로그램 실행 중 계속 유지되어야 하는 데이터를 저장하고 관리한다.
> 정적 변수와 전역 변수 성격의 데이터는 프로그램 전체에서 접근할 수 있으며, 특정 함수가 종료되어도 그 값이 사라지지 않고 유지된다.

**데이터 영역의 메모리 구조**
┌─────────────┐
│   Stack     │
├─────────────┤
│   Heap      │
├─────────────┤
│   Data      │ (BSS 영역 포함)
├─────────────┤
│   Code      │
└─────────────┘

---

## 데이터 영역의 장점
- 프로그램 전체에서 데이터를 쉽게 공유할 수 있다.
- 프로그램 실행 중 데이터가 지속적으로 유지된다.
- 함수 종료 후에도 값이 사라지지 않아 상태를 보존할 수 있다.
- 반복해서 사용하는 데이터를 매번 할당할 필요 없이 효율적으로 관리할 수 있다.

---

## 데이터 영역의 단점
- 불필요한 정적(Static) 변수 사용은 프로그램 종료 시까지 메모리를 차지하여 메모리 사용량을 증가시킬 수 있다.
- 프로그램 종료 전까지는 메모리가 자동으로 해제되지 않는다.
- 정적 변수의 남용은 객체 지향 설계를 해치고 코드 유지보수와 디버깅을 어렵게 만들 수 있다.
- 멀티스레드 환경에서 동시 접근 시 데이터가 오염될 수 있으므로 주의가 필요하다.

---

| 항목 | Data 영역 | Stack 영역 |
| :--- | :--- | :--- |
| **저장 내용** | 전역 변수, 정적 변수 | 지역 변수, 매개변수 |
| **메모리 관리** | 프로그램 종료 시 해제 | 함수 종료 시 자동 해제 |
| **생성 시점** | 프로그램 시작(로드) 시 | 함수 호출 시 |
| **제거 시점** | 프로그램 종료 시 | 함수 종료 시 |
| **데이터 유지** | 프로그램 종료까지 유지 | 함수 종료 시 삭제 |

---

## 활용 예시
- 게임 설정 값 및 환경 변수 저장
- 플레이어 수 및 인스턴스 개수 관리
- 프로그램 전체에서 공유하는 공통 유틸리티 데이터
- 정적 클래스의 데이터 관리

---

## 결론
**데이터 영역(Data Area)은 프로그램이 실행되는 동안 계속 유지되어야 하는 데이터를 저장하는 메모리 영역이다.**
> 주로 전역 변수와 정적 변수가 이 영역의 역할을 하며, 프로그램 시작 시 메모리가 할당되고 종료될 때까지 유지된다. 다만 메모리가 상주하므로 무분별한 사용은 지양하고 필요한 곳에만 효율적으로 활용해야 한다.
