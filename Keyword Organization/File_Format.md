# File Format

## 파일 포맷(File Format)이란?

파일 포맷(File Format)은 데이터를 저장하는 방식과 구조를 정의한 규칙이다.

파일 포맷에 따라 운영체제나 프로그램이 파일의 내용을 올바르게 읽고 사용할 수 있다.

일반적으로 파일 확장자(Extension)를 통해 파일의 종류를 구분한다.

---

## 파일 포맷의 특징

- 데이터의 저장 방식을 정의한다.
- 파일의 종류를 구분할 수 있다.
- 프로그램 간 데이터 교환이 가능하다.
- 확장자를 통해 파일 형식을 확인할 수 있다.

---

## 파일 포맷의 구성 요소

### 파일 이름(File Name)

사용자가 파일을 구분하기 위해 지정하는 이름이다.

예:

```
Report
```

---

### 확장자(Extension)

파일의 형식을 나타내는 부분이다.

예:

```
Report.pdf
```

여기서 `.pdf`가 확장자이다.

---

## 주요 파일 포맷

| 파일 포맷 | 설명 |
|-----------|------|
| .txt | 일반 텍스트 파일 |
| .pdf | 문서 파일 |
| .docx | Microsoft Word 문서 |
| .xlsx | Microsoft Excel 문서 |
| .pptx | Microsoft PowerPoint 문서 |
| .jpg | 이미지 파일 |
| .png | 투명 배경을 지원하는 이미지 파일 |
| .mp3 | 오디오 파일 |
| .mp4 | 동영상 파일 |
| .zip | 압축 파일 |
| .exe | 실행 파일 |
| .md | Markdown 문서 |

---

## 파일 포맷의 역할

- 데이터를 저장한다.
- 프로그램이 파일을 인식하도록 한다.
- 서로 다른 프로그램 간 데이터 공유를 가능하게 한다.
- 파일의 용도와 형식을 구분한다.

---

## 파일 포맷의 장점

- 파일의 종류를 쉽게 구분할 수 있다.
- 프로그램 간 호환성이 높아진다.
- 데이터를 효율적으로 저장할 수 있다.
- 다양한 프로그램에서 사용할 수 있다.

---

## 파일 포맷의 단점

- 프로그램마다 지원하는 포맷이 다를 수 있다.
- 호환되지 않는 포맷은 열 수 없다.
- 일부 포맷은 용량이 클 수 있다.
- 특정 프로그램이 필요할 수 있다.

---

## 활용 예시

- `.txt` : 메모 작성
- `.pdf` : 문서 공유
- `.docx` : 문서 작성
- `.png` : 이미지 저장
- `.mp4` : 영상 저장
- `.zip` : 여러 파일 압축
- `.md` : Markdown 문서 작성 및 GitHub 문서화

---

## 결론

파일 포맷은 데이터를 저장하고 관리하기 위한 규칙이다.

확장자를 통해 파일의 종류를 구분하며, 운영체제와 프로그램은 파일 포맷을 기반으로 데이터를 읽고 처리한다.
