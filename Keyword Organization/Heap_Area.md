# Heap Area

## 힙 영역(Heap Area)이란?

힙 영역(Heap Area)은 프로그램이 실행되는 동안 동적으로 메모리를 할당받아 사용하는 메모리 영역이다.

주로 객체(Object)와 동적으로 생성되는 데이터가 저장되며, C#에서는 `new` 키워드를 사용하여 생성한 객체가 힙 영역에 저장된다.

---

## 힙 영역의 특징

- 동적으로 메모리를 할당한다.
- 객체(Object)가 저장된다.
- 프로그램 실행 중 크기가 변경될 수 있다.
- 여러 메서드에서 공유하여 사용할 수 있다.
- 가비지 컬렉터(Garbage Collector)가 메모리를 관리한다.

---

## 힙 영역에 저장되는 데이터

### 객체(Object)

클래스를 이용하여 생성한 객체가 저장된다.

### 배열(Array)

배열은 힙 영역에 저장된다.

### 문자열(String)

문자열 객체는 힙 영역에 저장된다.

---

## 힙 영역의 역할

프로그램 실행 중 필요한 메모리를 동적으로 할당하여 객체를 저장하는 역할을 한다.

객체가 더 이상 사용되지 않으면 가비지 컬렉터(GC)가 자동으로 메모리를 회수한다.

---

## 힙 영역의 메모리 구조

```
┌─────────────┐
│    Code     │
├─────────────┤
│    Data     │
├─────────────┤
│    Heap     │
├─────────────┤
│    Stack    │
└─────────────┘
```

---

## C# 예시

```csharp
class Person
{
    public string Name;
}

class Program
{
    static void Main()
    {
        Person p = new Person();
    }
}
```

위 코드에서 `new Person()`으로 생성된 객체는 Heap 영역에 저장된다.

---

## 힙 영역의 장점

- 필요한 만큼 메모리를 사용할 수 있다.
- 객체를 여러 곳에서 공유할 수 있다.
- 동적인 데이터 관리에 적합하다.
- 메모리를 효율적으로 사용할 수 있다.

---

## 힙 영역의 단점

- 메모리 할당 속도가 Stack보다 느리다.
- 메모리 단편화(Fragmentation)가 발생할 수 있다.
- 가비지 컬렉션이 실행될 때 성능이 저하될 수 있다.
- 메모리 관리가 복잡하다.

---

## Heap 영역과 Stack 영역 비교

| 항목 | Heap 영역 | Stack 영역 |
|------|----------|-----------|
| 저장 내용 | 객체, 배열, 문자열 | 지역 변수, 매개변수 |
| 메모리 관리 | 가비지 컬렉터(GC) | 자동 관리 |
| 생성 시점 | 객체 생성 시 | 함수 호출 시 |
| 제거 시점 | GC가 회수 | 함수 종료 시 |
| 속도 | 상대적으로 느림 | 매우 빠름 |

---

## 활용 예시

- 클래스 객체 생성
- 배열 생성
- 문자열 객체 저장
- 게임 캐릭터 객체 생성
- 플레이어 데이터 관리

---

## 결론

Heap 영역은 객체와 동적으로 생성되는 데이터를 저장하는 메모리 공간이다.

C#에서는 `new` 키워드로 생성한 객체가 Heap 영역에 저장되며, 사용이 끝난 객체는 가비지 컬렉터(Garbage Collector)가 자동으로 메모리를 회수하여 관리한다.
