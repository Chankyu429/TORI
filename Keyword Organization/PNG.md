# PNG

![PNG](./image/4294118e-72d5-4d90-b1d8-fb3748398ad1)

## PNG(Portable Network Gaphics)란?

#### PNG는 디지털 이미지를 저장하기 위한 무손실 압축(Lossless Compression) 비트맵 이미지 파일 포맷이다.

---

W3C 표준으로 지정되어 있으며, 원본 이미지의 품질을 유지하면서 파일 크기를 줄일 수 있다. 특히 알파 채널(Alpha Channel)을 지원하여 투명 및 반투명 이미지를 완벽하게 표현할 수 있다.

- 확장자는 `.png`를 사용한다.

---

## PNG의 특징

- 무손실 압축 방식을 사용한다 : 이미지를 압축하고 재저장해도 원본 데이터의 손실이나 화질 저하가 없다.
- 알파채널(투명도)을 지원한다 : 단순히 투명/불투명만 구분하는 다른 포맷과 달리 0~255 단계의 반투명 효과를 표현할 수 있다.
- 텍스트 및 기하학적 그래픽에 최적화되어 있다 : 경계선이 뚜렷한 로고, 아이콘, 스크린샷 등을 저장할 때 JPEG보다 깨끗하게 표현된다.
- 인터레이싱(Interlacing)을 지원한다 : 웹 로딩 시 이미지가 위에서부터 순차적으로 나오지 않고, 흐릿한 상태에서 점차 선명해지는 방식을 지원한다.

---

## PNG의 동작 방식

#### PNG는 Deflate 압축 알고리즘(LZ77과 허프만 코딩의 결합)을 사용하여 무손실 압축을 수행한다.

압축 전 단계에서 픽셀 간의 차이를 예측하는 '필터링(Filtering)' 과정을 거치기 때문에 색상 변화가 적은 컴퓨터 그래픽이나 텍스트 이미지에서 압축 효율이 극대화된다.

---

## PNG의 장점

- 압축 및 디코딩 반복 시 화질 저하가 전혀 없다.
- 알파 채널을 통한 고품질 투명 배경을 지원한다.
- 웹 브라우저 및 대부분의 개발 환경(IDE, 게임 엔진 등)에서 기본 포맷으로 지원된다.

---

## PNG의 단점

- 사진과 같이 색상 스펙트럼이 연속적이고 복잡한 실사 이미지는 JPG보다 파일 용량이 훨씬 커진다.
- 애니메이션을 기본적으로 지원하지 않는다. (이를 해결하기 위해 APNG 포맷이 나왔으나 표준화가 느린 편이다.)

---

## PNG와 다른 포맷 비교 (JPG / BMP)

| 항목 | PNG | JPG(JPEG) | BMP |
|----|----|----|----|
| 압축 방식 | 무손실 압축 | 손실 압축 | 일반적으로 압축 없음 |
| 파일 크기 | 중간(그래픽에 효율적) | 작음(사진에 효율적) | 매우 큼 |
| 화질 | 원본 유지 | 압축률에 따라 손실 발생 | 원본 유지 |
| 투명 배경 | 지원 | 지원하지 않음 | 지원하지 않음 |
| 주요 용도 | 웹 UI, 로고, 아이콘, 게임 리소스 | 실사 사진, 웹 이미지 | 윈도우 시스템 내부 비트맵 데이터 |

---

## PNG 사용 예시

- 회사 로고
- 게임 UI
- 아이콘
- 웹사이트 이미지
- 캐릭터 스프라이트(Sprite)

## 결론

PNG는 무손실 압축 방식을 사용하는 이미지 파일 포맷이다.

원본 화질을 유지하면서 투명 배경을 지원하기 때문에 웹 개발, 게임 개발, UI 디자인, 그래픽 작업 등 다양한 분야에서 널리 사용된다.
