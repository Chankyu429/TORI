# Stack Area

## 스택 영역(Stack Area)이란?

스택 영역(Stack Area)은 함수 호출 시 생성되는 지역 변수와 매개변수(Parameter)가 저장되는 메모리 영역이다.

함수가 호출될 때 메모리가 할당되고, 함수가 종료되면 자동으로 메모리가 해제된다.

---

## 스택 영역의 특징

- 지역 변수가 저장된다.
- 매개변수(Parameter)가 저장된다.
- 함수 호출 시 메모리가 생성된다.
- 함수 종료 시 메모리가 자동으로 해제된다.
- LIFO(Last In First Out, 후입선출) 구조를 사용한다.

---

## 스택 영역에 저장되는 데이터

### 지역 변수(Local Variable)

메서드 내부에서 선언한 변수이다.

### 매개변수(Parameter)

함수 호출 시 전달되는 값이 저장된다.

### 함수 호출 정보

메서드의 실행 순서와 반환 위치(Return Address) 등의 정보가 저장된다.

---

## 스택 영역의 역할

함수 호출 과정에서 필요한 데이터를 저장하고 관리한다.

메서드가 종료되면 사용했던 메모리를 자동으로 반환하여 효율적으로 메모리를 관리한다.

---

## 스택 영역의 메모리 구조

```
┌─────────────┐
│   Stack     │
├─────────────┤
│    Heap     │
├─────────────┤
│    Data     │
├─────────────┤
│    Code     │
└─────────────┘
```

---

## C# 예시

```csharp
class Program
{
    static void Main()
    {
        int number = 10;
        Print(number);
    }

    static void Print(int value)
    {
        int result = value + 5;
        Console.WriteLine(result);
    }
}
```

위 코드에서

- `number`
- `value`
- `result`

변수는 모두 Stack 영역에 저장된다.

---

## 스택 영역의 장점

- 메모리 할당과 해제가 매우 빠르다.
- 함수 종료 시 자동으로 메모리가 해제된다.
- 메모리 관리가 간단하다.
- 접근 속도가 빠르다.

---

## 스택 영역의 단점

- 저장 가능한 메모리 크기가 제한적이다.
- 너무 많은 함수 호출은 Stack Overflow가 발생할 수 있다.
- 큰 데이터를 저장하기에 적합하지 않다.
- 함수가 종료되면 데이터가 사라진다.

---

## Stack 영역과 Heap 영역 비교

| 항목 | Stack 영역 | Heap 영역 |
|------|-----------|----------|
| 저장 내용 | 지역 변수, 매개변수 | 객체, 배열, 문자열 |
| 메모리 관리 | 자동 | 가비지 컬렉터(GC) |
| 생성 시점 | 함수 호출 시 | 객체 생성 시 |
| 제거 시점 | 함수 종료 시 | GC가 회수 |
| 속도 | 매우 빠름 | 상대적으로 느림 |

---

## 활용 예시

- 함수 호출
- 지역 변수 저장
- 매개변수 저장
- 재귀 함수 실행
- 메서드 실행 관리

---

## 결론

Stack 영역은 함수 호출과 지역 변수, 매개변수를 저장하는 메모리 공간이다.

함수가 호출되면 메모리가 생성되고, 함수가 종료되면 자동으로 메모리가 해제된다.

빠른 메모리 접근 속도를 제공하지만, 저장 공간이 제한되어 있으며 과도한 사용은 Stack Overflow를 발생시킬 수 있다.
